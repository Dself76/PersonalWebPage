SocioPedia - A Comprehensive Social Media Platform

Description:
SocioPedia is an innovative social media platform designed to connect users through various interactive features. It allows users to register and create profiles, participate in public and private voting on diverse topics, engage in discussions, and form groups for personalized interactions. Whether it’s voting on the latest political issues, choosing the best place to hang out with friends, or discussing current events, SocioPedia provides a dynamic and inclusive environment for all users.

Features:

User Registration and Profiles:

  • Sign Up Form: Users can create their profiles by providing essential details such as first name, last name, email, password, sex, country, state, and county.
  • Secure Login: User credentials are securely stored, and passwords are hashed to ensure privacy and security.

Voting and Polls:

  • Daily Topics: Users can vote on daily trending topics displayed on the homepage, such as political polls, sports predictions, and entertainment preferences.
  • Group Polls: Users can create groups (friends, family, business) and initiate private or public polls within these groups. This feature is ideal for making decisions like choosing a restaurant, planning outings, or any other group activities.
  • Custom Polls: Individuals can create custom polls about any topic of their interest and share them within their groups for voting.From where they want to eat dinner to politic to creating a simple fantasy league. This is desined to let people come up with their own polls or group decisions form simple to complex. At this point in the project i am focusing more on backend before the front end design, so please forgive the vagueness.

Discussion and Comments:

  • Comment Section: Users can post comments on various topics, engage in discussions, and share their views.
  • Trending Topics: A sidebar on the homepage highlights trending topics and current events, encouraging users to participate in ongoing discussions.

Group Creation and Management:

  • Personal Groups: Users can form personal groups with friends, family, or colleagues.
  • Group Voting: Within these groups, users can create and vote on polls, with options for public visibility or private voting.

Interactive User Interface:

  • Modern Design: The platform features a user-friendly interface with a modern design, ensuring a seamless user experience.
  • Responsive Layout: The layout is responsive and works well on both desktop and mobile devices.

Search and Autocomplete:

  • User Search: A search feature allows users to locate other users by entering usernames or keywords.
  • Autocomplete Function: The platform provides real-time suggestions as users type in the search bar, making it easier to find connections.

Backend and Database:

  • Flask Framework: The backend is developed using Flask, a lightweight and flexible Python web framework.
  • MySQL Database: User data and posts are stored in a MySQL database, ensuring efficient data management and retrieval.
  • RESTful APIs: The backend includes RESTful APIs for handling user registration, post submission, and data retrieval.

Technologies Used:

  • Frontend: HTML, CSS, JavaScript
  • Backend: Python, Flask
  • Database: MySQL
  • APIs: RESTful APIs
  • Additional Libraries: NLTK, SpaCy, Hugging Face Transformers for NLP tasks
  • Environment Management: dotenv for environment variables

Conclusion:

SocioPedia is more than just a social media platform; I envision the end product to be community where users can freely express their opinions, make collective decisions, and stay informed about current events. Its diverse set of features caters to the needs of different user groups, making it a versatile and engaging platform for everyone.It can set itself aside form other platforms by having a friendly easy to use interface to help groups come up with a decision(their vote) on their own topic, whether it be dinner, or who they and thier friends think will win a election to simple way to create a fantasy league.

Simple Password Manager

Project Name: Simple Password Manager

Language: C++

Project Overview:
The Simple Password Manager is a C++ application designed to securely store and manage user passwords. This project aims to provide a basic yet effective solution for handling password storage with a focus on simplicity and security. It demonstrates the fundamental concepts of file handling, encryption, and user authentication in C++.

Features:

Key Features:

  • Password Storage: Securely store passwords with associated usernames and services.
  • Encryption: Implement basic encryption techniques to protect stored passwords.
  • User Authentication: Ensure that only authorized users can access the stored passwords.
  • Command-Line Interface: Simple and intuitive command-line interface for user interactions.

Project Structure:

  • main.cpp: The main entry point of the application.
  • PasswordManager.cpp: Contains the core functionality of the password manager, including methods for adding, retrieving, and encrypting passwords.
  • PasswordManager.h: Header file defining the PasswordManager class and its methods.
  • PasswordManager.sln: Visual Studio solution file for the project.
  • PasswordManager.vcxproj: Visual Studio project file.
  • PasswordManager.vcxproj.filters: Filters file for organizing the project files in Visual Studio.
  • PasswordManager.vcxproj.user: User-specific settings for the Visual Studio project.

Development Environment:

  • Developed using Visual Studio, leveraging its robust development tools and debugging features.
  • The project is configured to build and run in a Debug configuration on a 64-bit Windows environment.

How to Run:

  • Open the PasswordManager.sln solution file in Visual Studio.
  • Build the project using the provided solution configuration.
  • Run the compiled executable to start the password manager.

Usage:

  • Add Password: Store a new password with the associated username and service name.
  • Retrieve Password: Retrieve stored passwords by providing the service name.
  • Encrypt/Decrypt: Basic encryption and decryption functionalities to secure stored passwords.

Learning Outcomes:

  • Understanding of file handling in C++.
  • Implementation of basic encryption techniques.
  • Experience with user authentication mechanisms.
  • Familiarity with developing and debugging C++ applications in Visual Studio.
